## Проект терминал

Терминал - консольный файловый менеджер, реализованный на C++ с использованием ООП. 
Программа позволяет выполнять базовые операции с файлами и директориями (CRUD - Create, Read, Update, Delete) 

## Возможности

- Навигация по директориям (cd, ls)
- Создание и удаление файлов и директорий
- Редактирование текстовых файлов (однострочных и многострочных)
- Переименование файлов и директорий
- Мягкое удаление (скрытие) с возможностью восстановления

## Команды
 _____________________________________________________________________________________
|      Команда     |                             Описание                             |
|------------------|------------------------------------------------------------------|
| `help`           | Показать список доступных команд                                 |                   
| `exit`           | Выйти из программы                                               |
| `ls`             |Показать содержимое текущей директории                            |
| `cd <name\|..>`  | Перейти в директорию или на уровень выше                         |
| `cat <name>`     | Показать содержимое файла                                        |
| `touch <name>`   | Создать новый файл (тип определится при первом редактировании)   |
| `mkdir <name>`   | Создать новую директорию                                         |
| `rm <name>`      | Скрыть файл или директорию (добавляет точку в начало имени)      |
| `restore <name>` | Восстановить ранее скрытую сущность                              |
| `mv <old> <new>` | Переименовать файл или директорию                                |
| `edit <name>`    | Редактировать содержимое файла                                   |
|__________________|__________________________________________________________________|

## Типы файлов

- **SingleLineFile** — однострочный текстовый файл
- **MultiLineFile** — многострочный текстовый файл
- **ZygoteFile** — файл-зигота (тип определяется при первом редактировании)

## Иерархия классов

Entry (базовый класс)

	Directory (директория)

	SingleLineFile (однострочный файл)

	MultiLineFile (многострочный файл)

	ZygoteFile (файл с неопределённым типом)

Terminal

## Ключевые архитектурные решения

1. Полиморфизм - все операции над сущностями выполняются через виртуальные методы базового класса 'Entry'

2. Мягкое удаление - вместо физического удаления файлы и директории скрываются (добавляется точка в начале имени)

3. Навигация - текущий путь храниться в векторе path

4. Отложенное определение типа - при создании файла создается 'ZygoteFile', тип определяется при первом редактировании
